Projet 1 – SAE 501 : Infrastructure TP Conteneurisée

SAE 501 – Mise en place d’une infrastructure de TP basée sur des conteneurs.

L’objectif du projet était de créer une plateforme permettant de déployer automatiquement des environnements de Travaux Pratiques accessibles via un navigateur web. Nous avons conçu une infrastructure complète utilisant Docker, Flask, MySQL et une interface en noVNC pour permettre aux étudiants de créer, démarrer, arrêter ou supprimer leurs machines virtuelles.

Ce que j’ai réalisé :

  • Mise en place d’un orchestrateur de conteneurs pour gérer plusieurs environnements.
  • Création d’images de conteneurs (Ubuntu, Kali Linux, Windows).
  • Développement d’une interface web en Flask + HTML/CSS/JS.
  • Système d’authentification sécurisé (mots de passe hachés, rôles admin/user).
  • Gestion dynamique des conteneurs Docker (création, démarrage, suppression).
  • Monitoring via logs utilisateur et supervision complète avec Portainer.

Fonctionnalités techniques :
  • Déploiement instantané de machines virtuelles via Docker.
  • Accès au bureau via navigateur grâce à noVNC.
  • Isolation complète des conteneurs.
  • Supervision système : CPU, RAM, nombre de conteneurs.

Projet 1 – Rapport automatique de sauvegardes quotidiennes

Contexte : L’entreprise IT2i effectuait manuellement chaque matin la vérification des sauvegardes Veeam/Nakivo de plusieurs dizaines de clients. Cela prenait beaucoup de temps et était source d’erreurs.

Mission confiée : Automatiser entièrement la génération et l’envoi d’un rapport PDF clair chaque jour.

Réalisation (octobre-novembre 2025) :

  • Création d’un script Java qui se connecte en SSH aux différents serveurs de sauvegarde.
  • Récupération des logs et statuts des jobs de sauvegarde de la veille.
  • Génération automatique d’un rapport PDF esthétique (avec iText 7) contenant :
    • Statut global (succès / échec / avertissement)
    • Liste détaillée des clients avec date, taille sauvegardée, durée
    • Graphiques simples (camembert taux de réussite)
  • Envoi automatique par e-mail via Gmail API avec Google Apps Script (pour contourner les restrictions SMTP classiques).
  • Planification quotidienne via cron (Linux) + tâche planifiée Windows.

Résultat :
Gain de 30 à 45 minutes par jour pour l’équipe technique + fiabilité accrue (plus d’oublis) + rapport beaucoup plus lisible pour les clients finaux quand on leur transmet.
